# Marc Lavoie | Cybersecurity Portfolio

Public GitHub Pages portfolio for Marc Lavoie. The site is built with plain HTML, CSS, and JavaScript only. It has no backend, database, login system, tracking scripts, API keys, or external JavaScript libraries.

## Files

- `index.html` - main portfolio page
- `styles.css` - responsive dark cybersecurity theme
- `script.js` - mobile navigation and active-section highlighting
- `SECURITY-CHECKLIST.md` - public publishing safety checklist

## Local Preview

Open `index.html` in a browser to preview the site locally. No build step is required.

## GitHub Pages Deployment

This repository is intended to be published as:

`https://MarcCyberLavoie.github.io`

To deploy with GitHub Pages:

1. Push these files to the `main` branch of `MarcCyberLavoie.github.io`.
2. Open the repository on GitHub.
3. Go to **Settings**.
4. Open **Pages**.
5. Under **Build and deployment**, choose **Deploy from a branch**.
6. Select the `main` branch and the repository root folder.
7. Save the Pages settings.
8. Wait for GitHub Pages to finish publishing.

## Updating Content

Replace placeholder project links, screenshots, training entries, resume link, and contact links only after reviewing `SECURITY-CHECKLIST.md`.

Use sanitized screenshots and synthetic sample data. Do not publish real logs, credentials, private contact information, internal IP addresses, cloud project IDs, or private school or company details.

## Security Notes

Before every public update:

- Review changed files for secrets and private data.
- Review screenshots for names, emails, IP addresses, project IDs, hostnames, tokens, and account details.
- Confirm repository history does not contain sensitive material.
- Keep the site static and dependency-free unless there is a clear reason to change it.
